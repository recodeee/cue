## Task-10 deferral — rationale (user-approved 2026-04-17)

Task 10 (1.x baseline capture) is DEFERRED to Task 35/44. The following sequence of discoveries led to the decision:

1. **Plan-design gap found at smoke-test attempt 1:** Case `diff.patch` files reference source files (e.g. `src/interceptors/auth.ts`) that do not exist in the Hydra repo. Plan assumed submodule-based sample repos, but those are Task 35 (Part 5).

2. **First bug-fix pass:** `bench/runner/invoke_hydra_1x.py` had 2 blockers — `--cwd` is not a valid `claude` CLI flag, and `apply_diff` used `check=False` (silent failure on patch apply). Fixed in `e550beb`. Fail-loud change saved ~$0.60–3 that would have been spent on base-code-only analyses.

3. **Inline mini-workspace pivot:** Each case got a `workspace/` dir with minimal reconstructed base files. Harness rewritten to copy→git-init→commit→apply→invoke. Plus 5/5 `diff.patch` files had corrupt hunk-count headers — fixed in `89ba1cb`.

4. **Smoke-test attempt 2 (2026-04-17 14:52) — timeout:**
   ```
   subprocess.TimeoutExpired: Command '['claude', '--print', '/hydra this']'
       timed out after 600 seconds
   ```
   Suspected causes: (a) Hydra 1.x deep-mode runs 3–6 Codex advisors sequentially (SKILL.md §336), exceeding the 10-min timeout, (b) nested `claude --print` hangs on permission prompts without `--dangerously-skip-permissions` or `--allowed-tools` allowlist.

5. **Strategic evaluation by Opus advisor agent:** "Die Baseline-in-Part-2 ist prozedurale Absicherung, keine wissenschaftliche." Core finding: inline-workspace baseline is **not kommensurabel** with Task 44's regression-check, which will run against Task-35 submodules. Capturing now would be sunk cost; Task 44 would have to remeasure against real repos anyway. Recommended deferring Task 10 rather than investing further in workarounds.

**Root cause:** Task 10 position in plan is wrong. It depends on Task 35 but was scheduled 25 tasks earlier. All downstream issues (corrupt hunks, nonexistent files, harness timeouts, permission chains) are symptoms.

**Fix faithful to CLAUDE.md "Root Causes fixen, keine Workarounds":** Defer Task 10 to Task 44 sub-step where it belongs.

### Important (fix before Task 44)

**I-1: `prepare_case_workspace()` leaks tmpdir on git init/add/commit failure.** `bench/runner/invoke_hydra_1x.py:32-41`. Only `git apply` exception is caught and cleans up `scratch`. Earlier git steps abandon the tmpdir on failure. Fix: wrap body in `try/finally` or `contextlib.ExitStack`.

**I-2: `extract_from_report()` raises uncaught `yaml.ScannerError` on malformed frontmatter.** `bench/runner/extract_findings.py:16`. Will crash entire bench run rather than skipping the case. Fix for Task 44: catch `yaml.YAMLError`, log warning, return `[]` with record that the report was unparseable.

**I-3: Ground-truth line ranges inaccurate; masked by `tol=10` over-generosity.** Case 01 declares `"lines":"14-28"` but actual block is `14-18` in a 21-line file. `_ranges_overlap()` tolerance so wide that `"14-28" vs "1-5"` matches. Fix before Task 44: either tighten tolerance + correct all 5 ground-truth ranges, or document intentional generosity via named constant.

**I-4: `HYDRA_TIMEOUT_S` not env-overridable.** `bench/runner/invoke_hydra_1x.py:18`. 600s constant is the only knob; smoke-test confirmed insufficient. Fix: `int(os.environ.get("HYDRA_TIMEOUT_S", "600"))`.

### Minor (nice-to-have)

**M-1: `write_baseline()` uses `s.__dict__` instead of `dataclasses.asdict()`.** `run_bench.py:57`. Works identically now; `asdict()` is forward-safe if `CaseScore` ever gains nested fields.

**M-2: Empty-PR-description path in `build_echo_prompt` (Task 15 plan).** Empty-string join produces trailing double-newline separator. Harmless but worth `filter(None, [...])` guard when implementing.

**M-3: `extract_from_report` tests cover only happy path.** Missing: empty `top_actions`, `top_actions: null`, action with no `file` key.

**M-4: `test_prepare_case_workspace_missing_workspace_dir_raises` uses direct attribute mutation.** `patch("bench.runner.invoke_hydra_1x.CASES_DIR", ...)` would be more idiomatic. Acceptable for now.
